Data mining and visualization

— social media orientated

Due to the volume of data produced on social media is extremely huge, we expect to implement the system on the basis of easily fetching popular topics and handily extracting subtle information to help people knowing what they are talking about on social media. We propose to achieve the project from the following aspects:
        ● Ability to process real-time data and update the model periodically.
        ● Topics are integrated with the past and current data in order to observe the trend of the topic.
        ● The system capability can process volume data under the stable situation to guarantee efficiency and sensitivity of emerging topics.
        ● Visualization of statistical results.

Details

The Data mining and visualization system is a web-based product, and the front-end is experimental setup by computer language HTML, CSS, JavaScript and Nodejs. The system is available to run on any PC accompanying with well-setup back-end environment which installed Flask, Tweepy, and MongoDB.
The implementation details can be divided mainly to 5 units based on the functions, which GUI, API calling, line chart, pie chart and 3D global earth.

1) GUI


The elements (input, line charts, pie charts, 3D global) of the webpage is directly hypertext reference Bootstrap (the world’s most popular front-end component library) stylesheet and Echarts (A Declarative Framework for Rapid Construction of Web-based Visualization). All the components are placed by using existing template from GitHub.
There are 2 webpage in this system, one is search page, and the other is using keywork to extract data to display data in specific diagrams, e.g. line chart, pie chart and 3D global.


2) API calling


API calling is designed to retrieved JSON data from local back-end through URL by using Ajax GET request.For the interface, the API calling program will provide keywordData with one parameter keyword.

3) Line chart


The response data returns from API is in JSON format. In order to fit in the code provided by Echarts Line chart, the data should be formatted to xAxis (represent the week from Mon. to Sun.) and yAxis (represent the total amount of a specific tweets all around the world).
To be specific the data in line.data is an array [‘Mon’, ‘Tue’, ‘Wed’, ‘Thu’, ‘Fri’, ‘Sat’, ‘Sun’] and the line.vaule is the according amount of tweets which are [2919, 2238, 2453, 3479, 3397, 2429, 2627].

4) Pie chart


The return data form backend of each continent is
Pie: [{value: 2488, name: 'Asia'},
{value: 4394, name: 'Africa'},
{value: 8998, name: 'NorthAmerica'},
{value: 1863, name: 'SouthAmerica'},
{value: 1799, name: 'Europe'}]
To put the above data into Pie chart, simply provide the pie to include all the return value

5) 3D global


The data return is global: [{longitude, latitude, density}, …], before directly put the data into 3D global, the data need be reformatted as the Echarts request.
Then the data can be processed by 3D global to pin the each light point.
